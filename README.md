# BUMP OF CHICKEN 天体観測録(サンプルサイト)

ご要望いただいた仕様を反映した、フレームワーク不使用(HTML / CSS / JS のみ)のファンサイトのテンプレートです。
git・VSCode がある環境であれば、そのまま `git init` してすぐに編集を始められます。

## ファイル構成

```
bump-site/
├── index.html        ← ページ全体の骨組み(ヘッダー・サイドバー・本文・フッター)
├── css/style.css      ← デザイン(配色・フォント・レイアウト)
├── js/data.js          ← コンテンツの中身(メンバー・アルバム・曲)※ここを書き換えて運用します
├── js/app.js            ← 表示ロジック(タブ切り替え・検索・ページネーション)
└── images/
    ├── members/        ← メンバー写真を入れる場所(任意)
    └── albums/          ← アルバムジャケット画像を入れる場所(任意)
```

## ローカルでの確認方法

`index.html` を直接ブラウザで開くだけで動作しますが、`fetch` 等を使う場合に備えて
簡易サーバーで見るのがおすすめです。

```bash
cd bump-site
python3 -m http.server 8080
# ブラウザで http://localhost:8080 を開く
```

## 実装した仕様と、その対応箇所

| ご要望 | 対応箇所 |
|---|---|
| 右側に最近のアップロード曲(最大10件)とリンク | `index.html` の `<aside class="log-panel">` / `js/app.js` の `renderRecentList()` |
| 中央にバンドの簡単な説明 + メンバー写真 | ホームタブ `viewHome()`(メンバー写真は `#members` タブへのリンクになっています) |
| 検索バー → 全曲カード一覧 → 10曲ごとにページ分割 | `js/app.js` の `renderCatalog()` / `renderPagination()`(この検索+一覧は**全タブ・曲詳細ページでも常に表示される共通パーツ**にしています) |
| 上部にタイトル + タブ(ホーム/メンバー/アルバム/ライブ定番曲) | `index.html` の `<header>` / `js/app.js` の `router()` |
| メンバータブ → バンド紹介+各メンバーの説明と画像 | `viewMembers()` |
| アルバムタブ → 各アルバム画像+特徴 | `viewAlbum()` |
| ライブ定番曲タブ → 該当曲一覧 → クリックで曲詳細へ | `viewLive()`(`data.js` の `liveStandard: true` で判定) |
| 曲詳細ページ:タイトル/タブ/最近のアップロード/検索バー/一覧は固定、曲名→サブタイトル→基本情報→YouTube→箇条書き | `viewSongDetail()` |

ヘッダー・右サイドバー(観測ログ)・下部の検索+全曲一覧は、`index.html` 側に一度だけ配置し、
タブやページが切り替わっても消えない「共通パーツ」にしています。中央の `#view-root` 部分だけが
ホーム/メンバー/アルバム/ライブ定番曲/曲詳細で切り替わる仕組みです(ページ遷移なしで瞬時に切り替わります)。

## コンテンツの編集方法(`js/data.js`)

- **メンバー**:`MEMBERS` 配列の `bio` を書き換え。写真を使う場合は `images/members/` に画像を置き、
  `photo: "images/members/fujihara.jpg"` のように1行追加すると、丸いアイコン部分が写真に切り替わります(未指定なら頭文字アイコンのまま)。
- **アルバム**:`ALBUMS` 配列に `photo: "images/albums/jupiter.jpg"` を追加するとジャケット画像が表示されます。
- **曲**:`SONGS` 配列の1曲が1オブジェクトです。
  - `liveStandard: true` にすると「ライブ定番曲」タブに表示されます。
  - `youtubeId` に YouTube 動画のID(URLの `watch?v=` の後ろの11文字)を入れると、曲詳細ページに動画が埋め込まれます。
  - `date` を更新すると「観測ログ(最近のアップロード)」の並び順に反映されます。

> **重要**:現在入っているメンバー紹介文・アルバム紹介文・曲の説明文は、サイトの動作確認用のサンプル文章です。
> 実際の人物写真やメンバー紹介文、歌詞・記事の引用などは著作権・肖像権にあたるため、このテンプレートには含めていません。
> 公開する際は、ご自身で用意した文章・画像、または公式が許可している素材に置き換えてご利用ください。

## デザインについてのアドバイス(天体観測モチーフ)

「おしゃれな天体デザイン」というご要望に対して、単に背景を暗くして星を散らすだけでは
よくあるテンプレート感が出てしまうため、**「観測ログ」という一本の世界観**で全体を統一しました。

- **配色**:背景は深い宇宙の紺(`#0a0d1c`)。アクセントは星雲のすみれ色(`#8b7bf2`)と
  彗星の金色(`#e8b954`)の2色を使い分けています。すみれ色はホバーやリンクの「光」、
  金色は現在地を示すタブ下のラインや検索フォーカスなど「導線」として役割を分けると、
  色が多くてもごちゃつきません。
- **フォント**:見出しは「Shippori Mincho(明朝)」、本文は「Zen Kaku Gothic New」、
  日付やカタログ番号などのメタ情報は「JetBrains Mono」の3役構成にしています。
  明朝体の見出しが「観測ログ」らしい少し格式のある雰囲気を作り、等幅フォントの番号表記
  (`No.001` など)が"星のカタログ"らしさを出してくれます。
- **構造のモチーフ**:タブの下にある線を「軌道」、現在のタブを軌道上の「彗星」が示す、
  という小さな仕掛けを入れています。曲カードにも `No.001` のようなカタログ番号を振り、
  サイドバーの更新履歴を「観測ログ」と呼ぶなど、機能名そのものに世界観を持たせると、
  装飾を増やさなくても"おしゃれさ"が出ます。
- **動きは最小限に**:星の点滅とタブ下のラインの移動だけにとどめています。天体テーマは
  アニメーションを増やしたくなりますが、動きが多いほどページが重く、AI生成感も出やすいため、
  「効果は一箇所に集中させる」のがコツです。
- **発展アイデア**:お好みで、アルバムジャケットを実際の正方形画像に差し替えると一気に
  本格的になります。また、ライブ定番曲ページに「セットリストの並び順」のような演出
  (番号を大きくする、軌道線でつなぐ等)を加えると、ライブ感がさらに強まります。
