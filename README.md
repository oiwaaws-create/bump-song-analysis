# BUMP OF CHICKEN 天体観測録(サンプルサイト)

ご要望いただいた仕様を反映した、フレームワーク不使用(HTML / CSS / JS のみ)のファンサイトのテンプレートです。
git・VSCode がある環境であれば、そのまま `git init` してすぐに編集を始められます。

> 曲に「概要・解釈リンク・好きな歌詞」を追加する作業だけを1ファイルにまとめたガイドを
> 別途用意しています → [`song-commentary-guide.md`](./song-commentary-guide.md)

## ファイル構成

```
bump-site/
├── index.html          ← ページ全体の骨組み(ヘッダー・サイドバー・本文・フッター)
├── css/style.css        ← デザイン(配色・フォント・レイアウト)
├── js/
│   ├── data.js           ← バンド紹介・メンバー・アルバム・画像パスなど(曲データ以外)
│   ├── app.js             ← 表示ロジック(タブ切り替え・検索・ページネーション)
│   └── songs/              ← 曲データ。1曲1ファイル(例: tentai-kansoku.js)
│       ├── tentai-kansoku.js
│       ├── k.js
│       └── ...(曲ごとに1ファイル)
└── images/
    ├── members/        ← メンバー集合写真などを入れる場所(任意)
    └── albums/          ← アルバムジャケット画像を入れる場所(任意)
```

曲が増えるたびに大きな1ファイルを編集するのを避けるため、**曲データだけは `js/songs/` の中に1曲1ファイルで分けています**。
詳しい追加手順は下の「曲を1曲追加する手順」を参照してください。

## ローカルでの確認方法

`index.html` をダブルクリックして直接ブラウザで開くだけで動作します(サーバー不要)。
編集しながら自動リロードしたい場合は、VSCodeの拡張機能「Live Server」を入れて
`index.html` を右クリック→「Open with Live Server」が手軽です。

## 実装した仕様と、その対応箇所

| ご要望 | 対応箇所 |
|---|---|
| 右側に最近のアップロード曲(最大10件)とリンク | `index.html` の `<aside class="log-panel">` / `js/app.js` の `renderRecentList()` |
| 中央にバンドの簡単な説明 + 写真 | ホームタブ `viewHome()`(バンド紹介の直下に1枚の画像のみを表示。メンバー個別の写真は表示しません) |
| 検索バー → 全曲カード一覧 → 10曲ごとにページ分割 | `js/app.js` の `renderCatalog()` / `renderPagination()`(この検索+一覧は**全タブ・曲詳細ページでも常に表示される共通パーツ**にしています) |
| 上部にタイトル + タブ(ホーム/メンバー/アルバム/ライブ定番曲) | `index.html` の `<header>` / `js/app.js` の `router()` |
| タブ直下にパンくずリスト | `index.html` の `<nav id="breadcrumbs">` / `js/app.js` の `renderBreadcrumbs()`(現在地に応じて「ホーム › アルバム › jupiter › 天体観測」のように自動生成されます) |
| メンバータブ → バンド紹介+各メンバーの説明文(個別写真なし)+4人の集合写真1枚 | `viewMembers()`(各メンバー紹介の上に集合写真を1枚表示し、紹介カード自体には写真を入れていません) |
| アルバムタブ → 各アルバム画像+特徴。アルバムを押すと説明ページへ。そのページに曲一覧があり、曲を押すと曲詳細へ | `viewAlbum()`(一覧)→ `viewAlbumDetail()`(詳細・収録曲一覧)→ `viewSongDetail()`(曲詳細) |
| ライブ定番曲タブ → 該当曲一覧 → クリックで曲詳細へ | `viewLive()`(各曲ファイル `js/songs/*.js` の `liveStandard: true` で判定) |
| 曲詳細ページ:タイトル/タブ/最近のアップロード/検索バー/一覧は固定、曲名→サブタイトル→基本情報→YouTube→箇条書き | `viewSongDetail()` |

ヘッダー・右サイドバー(観測ログ)・下部の検索+全曲一覧は、`index.html` 側に一度だけ配置し、
タブやページが切り替わっても消えない「共通パーツ」にしています。中央の `#view-root` 部分だけが
ホーム/メンバー/アルバム/ライブ定番曲/曲詳細で切り替わる仕組みです(ページ遷移なしで瞬時に切り替わります)。

## 文言を修正する方法

| 直したい内容 | 編集するファイル | 編集する項目 |
|---|---|---|
| バンドの紹介文(ホーム・メンバータブ上部) | `js/data.js` | `BAND_INTRO` |
| メンバーの紹介文 | `js/data.js` | `MEMBERS` 配列の各 `bio` |
| アルバムの特徴・説明文 | `js/data.js` | `ALBUMS` 配列の各 `desc` |
| 曲のサブタイトル・作詞作曲・リリース日 | `js/songs/<曲のファイル>.js` | `subtitle` / `lyricist` / `composer` / `releaseDate` |
| 曲の箇条書き説明 | `js/songs/<曲のファイル>.js` | `bullets`(配列。1項目=1行) |

どの項目も `"..."` の中の日本語を書き換えるだけです。保存してブラウザを再読み込み(Live Server なら自動)すれば反映されます。

## 画像を入れる方法

画像ファイルは `images/` フォルダに置き、データ側にその「パス」を文字列で書くと表示されます。挿入できる場所は次の3つです。

| 画像 | 編集するファイル | 編集する項目 | 例 |
|---|---|---|---|
| ホーム画面・説明文の直下の1枚 | `js/data.js` | `HOME_PHOTO` | `const HOME_PHOTO = "images/home-photo.jpg";` |
| メンバータブ・紹介一覧の上の集合写真1枚 | `js/data.js` | `MEMBERS_GROUP_PHOTO` | `const MEMBERS_GROUP_PHOTO = "images/members/group.jpg";` |
| アルバムジャケット(アルバムごと) | `js/data.js` | `ALBUMS` 配列の対象アルバムに `photo` を追加 | `photo: "images/albums/jupiter.jpg"` |

手順はどれも同じです。

1. 画像ファイルを `images/`(または `images/members/` `images/albums/`)に置く
2. 上の表の項目に、そのファイルへの相対パスを文字列で入れる
3. 保存してブラウザを再読み込み

> 曲詳細ページには静止画ではなく YouTube 動画を埋め込む仕様なので、曲ごとの画像欄はありません
> (動画は各曲ファイルの `youtubeId` に動画IDを入れると表示されます)。

> **著作権・肖像権について**:メンバーの実写真やアルバムジャケットは、ご自身で撮影・用意したもの、
> または公式が使用を許可している素材を使ってください。サンプルの紹介文・説明文も同様に、
> 公開前に実際の内容へ書き換えることをおすすめします。

## 曲を1曲追加する手順

曲データは `js/songs/` の中に1曲1ファイルで入っているので、**既存のファイルを編集せずに**追加できます。

1. `js/songs/` の中の好きな既存ファイル(例: `tentai-kansoku.js`)をコピーする
2. コピーしたファイルの名前を、新しい曲用のID(半角英数字、ハイフンOK)に変更する
   例: `new-song.js`
3. ファイルの中身を書き換える

   ```js
   /* 曲名(表示用コメント) */
   SONGS.push({
     "id": "new-song",            // ファイル名と揃えておくと管理しやすいです
     "catNo": "No.026",           // カタログ番号(表示用。次の番号を割り振ってください)
     "title": "曲名",
     "subtitle": "サブタイトル",
     "album": "アルバム名",         // ALBUMS の title と完全一致させると、アルバム詳細ページの収録曲一覧に出ます
     "lyricist": "藤原基央",
     "composer": "藤原基央",
     "releaseDate": "2026-06",
     "youtubeId": "",              // YouTube動画ID(11文字)。空なら「動画未設定」と表示されます
     "liveStandard": false,        // true にすると「ライブ定番曲」タブに表示されます
     "date": "2026-06-23",         // 新しい順に「観測ログ(最近のアップロード)」に出ます
     "bullets": [
       "箇条書きの説明その1。",
       "箇条書きの説明その2。"
     ]
   });
   ```

4. `index.html` を開き、`<!-- 曲データ(1曲1ファイル)。新しい曲を追加するときは、ここに1行追加してください -->`
   というコメントのすぐ下に、次の1行を追加する(他の曲と同じ並びの場所でOKです)

   ```html
   <script src="js/songs/new-song.js"></script>
   ```

5. 保存してブラウザを再読み込みすれば、検索結果・全曲一覧・観測ログ・(`liveStandard:true` なら)ライブ定番曲一覧に反映されます

新しい曲を追加するたびに触るのは「新しいファイル1つ」+「`index.html` への1行追加」だけで、
既存の曲データのファイルには一切触れません。

曲には、`interpretation`(外部の解釈記事リンク)と `favoriteLyrics`(「1番」「2番」「Cメロ+大サビ」のような
ブロックに分けて、好きな歌詞+コメントのセットを並べる機能)という項目も追加できます(任意。現時点では
「アカシア」のみ設定しています)。
この2項目だけを追加・編集する手順は [`song-commentary-guide.md`](./song-commentary-guide.md) にまとめています。

## デザインについてのアドバイス(天体観測モチーフ)

「おしゃれな天体デザイン」というご要望に対して、単に背景を暗くして星を散らすだけでは
よくあるテンプレート感が出てしまうため、**「観測ログ」という一本の世界観**で全体を統一しました。

- **配色**:背景は深い宇宙の紺(`#0a0d1c`)。アクセントは星雲のすみれ色(`#8b7bf2`)と
  彗星の金色(`#e8b954`)の2色を使い分けています。すみれ色はホバーやリンクの「光」、
  金色は現在地を示すタブ下のラインや検索フォーカスなど「導線」として役割を分けると、
  色が多くてもごちゃつきません。
- **フォント**:見出しは「Shippori Mincho(明朝)」、本文は「Zen Kaku Gothic New」、
  日付やカタログ番号などのメタ情報は「JetBrains Mono」の3役構成にしています。
  明朝体の見出しが「観測ログ」らしい少し格式のある雰囲気を作り、等幅フォントの番号表記
  (`No.001` など)が"星のカタログ"らしさを出してくれます。
- **構造のモチーフ**:タブの下にある線を「軌道」、現在のタブを軌道上の「彗星」が示す、
  という小さな仕掛けを入れています。曲カードにも `No.001` のようなカタログ番号を振り、
  サイドバーの更新履歴を「観測ログ」と呼ぶなど、機能名そのものに世界観を持たせると、
  装飾を増やさなくても"おしゃれさ"が出ます。
- **動きは最小限に**:星の点滅とタブ下のラインの移動だけにとどめています。天体テーマは
  アニメーションを増やしたくなりますが、動きが多いほどページが重く、AI生成感も出やすいため、
  「効果は一箇所に集中させる」のがコツです。
- **発展アイデア**:お好みで、アルバムジャケットを実際の正方形画像に差し替えると一気に
  本格的になります。また、ライブ定番曲ページに「セットリストの並び順」のような演出
  (番号を大きくする、軌道線でつなぐ等)を加えると、ライブ感がさらに強まります。

## 今後、公開する際の手順

### 公開前のチェックリスト

- [ ] `js/data.js` の `BAND_INTRO` / `MEMBERS[].bio` / `ALBUMS[].desc` を実際の文章に書き換えた
- [ ] `js/songs/*.js` の各曲の `subtitle` `bullets` などを実際の内容に書き換えた
- [ ] `HOME_PHOTO` `MEMBERS_GROUP_PHOTO` `ALBUMS[].photo` に使う画像を、自分で用意した(または公式が許可した)ものに差し替えた、または使わないなら空のままにした
- [ ] 主要な曲の `youtubeId` を設定した(空のままでも動作しますが、「動画未設定」の表示になります)
- [ ] ローカルでホーム/メンバー/アルバム/ライブ定番曲タブ、検索、ページ送り、曲詳細、アルバム詳細、パンくずリンクをひと通り操作して確認した

### 1. GitHubへpushする

普段使っている手順のままでOKです。

```powershell
cd C:\Users\TOMOKI\Documents\bump-song-analysis
git add .
git commit -m "コンテンツを更新"
git push
```

### 2. GitHub Pagesで公開する(初回のみ設定)

1. GitHubのリポジトリページを開く
2. **Settings** → 左メニューの **Pages**
3. "Build and deployment" の Source を **Deploy from a branch** にする
4. Branch を `main` / フォルダを `/ (root)` にして **Save**
5. 1〜数分後、ページ上部に公開URL(`https://oiwaaws-create.github.io/bump-song-analysis/` のような形)が表示されます

これ以降は、`git push` するだけで自動的に公開ページが更新されます(設定のやり直しは不要です)。

### 3. 公開後の確認

- 公開URLにスマホでアクセスして、レイアウト崩れがないか確認する
- 一部のブラウザのプライベートモードなどで、想定外のキャッシュが残っていないか確認する
- 曲を追加・編集したら、`git push` を忘れずに(push するまで公開ページには反映されません)
