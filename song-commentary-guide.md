# 曲解説の追加方法(コーディング〜リリースまで)

このファイル1つで、「曲解説(概要・解釈リンク・好きな歌詞)」を1曲分追加して公開するまでの
作業がすべて完結します。「アカシア」を例にしていますが、どの曲でも同じ手順です。

対象ファイル: `js/songs/<曲のID>.js`(例: `js/songs/acacia.js`)

---

## 0. 全体の流れ

1. 曲ファイルをコーディングする(新曲なら新規作成、既存曲なら追記)
2. `index.html` に読み込み用の1行を追加する(新曲のときだけ)
3. ローカルのブラウザで表示を確認する
4. Gitでコミット・プッシュする
5. GitHub Pagesで公開されたことを確認する

---

## 1. コーディングする

### 1-A. 新しい曲を追加する場合

1. `js/songs/` フォルダの中の好きな既存ファイル(例: `acacia.js`)をコピーする
2. ファイル名を新しい曲のID(半角英数字・ハイフンOK)に変更する
   例: `tentai-kansoku.js`
3. 中身を以下のテンプレートに沿って書き換える

### 1-B. 既存の曲に「曲解説」だけ追記する場合

`js/songs/<曲のID>.js` を開き、既存の `bullets` の後ろに、下のテンプレートの
`"interpretation"` と `"favoriteLyrics"` の部分だけをコピーして追記します
(`interpretation` や `favoriteLyrics` が不要な曲は、書かなくてもOKです)。

### テンプレート(全項目)

```js
/* 曲名(表示用コメント) */
SONGS.push({
  "id": "new-song",                // ファイル名と揃えると管理しやすい
  "catNo": "No.026",               // カタログ番号。次の番号を割り振る
  "title": "曲名",
  "subtitle": "サブタイトル",
  "album": "アルバム名",            // ALBUMS の title と完全一致させると、アルバム詳細ページの収録曲一覧に出る
  "lyricist": "藤原基央",
  "composer": "藤原基央",
  "releaseDate": "2026-06",
  "youtubeId": "",                  // YouTube動画ID(11文字)。空なら「動画未設定」と表示される
  "liveStandard": false,            // true なら「ライブ定番曲」タブに表示される
  "date": "2026-06-23",             // 新しい順に「観測ログ(最近のアップロード)」に出る

  // ── ここが「曲解説」の本体 ──────────────────────────

  "bullets": [
    // 概要(ざっくり曲解説)。1項目=1行。何行でもOK
    "曲の背景や特徴をここに書く。",
    "ライブでの様子や聴きどころなど。"
  ],

  // 外部の解釈記事へのリンク。不要なら、この interpretation のブロックごと削除してOK
  "interpretation": {
    "note": "一言コメント(例: この人の解釈が面白い、など)",
    "url": "https://example.com/...",
    "label": "解釈記事を読む"
  },

  // 好きな歌詞。「1番」「2番」「Cメロ+大サビ」のようなブロックに分け、
  // 各ブロックの中に quotes+comment のセットを何個でも並べられる。
  // 不要なら favoriteLyrics ごと削除、または空配列 [] にしてOK
  "favoriteLyrics": [
    {
      "label": "1番",
      "items": [
        {
          "quotes": [
            "ここに歌詞の引用(数行)を入れる"
          ],
          "comment": "その引用に対するコメント・解釈をここに書く"
        },
        {
          "quotes": [
            "別の引用をここに入れる"
          ],
          "comment": "この引用へのコメントをここに書く"
        }
      ]
    },
    {
      "label": "2番",
      "items": [
        {
          "quotes": ["ここに2番の引用を入れる"],
          "comment": "コメントをここに書く"
        }
      ]
    },
    {
      "label": "Cメロ+大サビ",
      "items": [
        {
          "quotes": ["ここにCメロ+大サビの引用を入れる"],
          "comment": "コメントをここに書く"
        }
      ]
    }
  ]
});
```

ブロックの名前(`label`)は曲の構成に合わせて自由に変えられます(例: 「イントロ」「Bメロ」など)。
ブロックの中の `items` には、`quotes`(1つ以上の引用)+ `comment`(その引用へのコメント)の
セットを好きなだけ追加できます。まだコメントを書いていないブロックは `"items": []` のままでOKです
(「このブロックはまだ未入力です」と表示されます)。

### 歌詞の引用について(重要)

`favoriteLyrics` の `quotes` には、**実際の歌詞をご自身の手で入力**してください。
歌詞は著作権で保護されている表現そのものなので、AIが代わりに入力・生成することは避けています。
コメント文(`comment`)はご自身の感想・解釈なので、そのまま書いて問題ありません。

---

## 2. `index.html` に読み込み用の1行を追加する(新しい曲ファイルを作った場合のみ)

既存の曲に追記しただけの場合は、この手順は不要です(スキップしてください)。

`index.html` を開き、コメント
`<!-- 曲データ(1曲1ファイル)。新しい曲を追加するときは、ここに1行追加してください -->`
のすぐ下に、次の1行を追加します(他の曲と同じ並びでOK)。

```html
<script src="js/songs/new-song.js"></script>
```

---

## 3. ローカルで表示を確認する

`index.html` をダブルクリックしてブラウザで開く(または Live Server で開く)。

確認するポイント:

- [ ] 「全曲一覧」または「ライブ定番曲」一覧から、その曲の詳細ページに飛べる
- [ ] 曲解説(`bullets`)が箇条書きで表示されている
- [ ] `interpretation` を入れた場合、「◇ 解釈」のリンクが新しいタブで開く
- [ ] `favoriteLyrics` を入れた場合、「◇ (運営者名)の好きな歌詞」に、ブロック名(1番/2番など)の見出しごとに
      歌詞の引用(明るい色の枠)とコメントが表示されている
- [ ] 歌詞のプレースホルダー(`※ここに...入力してください`)が、実際の歌詞に置き換わっている

---

## 4. Gitでコミット・プッシュする

```powershell
cd C:\Users\TOMOKI\Documents\bump-song-analysis
git add .
git commit -m "曲解説を追加: 曲名"
git push
```

---

## 5. 公開されたことを確認する

GitHub Pagesの公開URL(例: `https://oiwaaws-create.github.io/bump-song-analysis/`)にアクセスし、
追加した曲の詳細ページが表示されることを確認します。
push してから公開ページに反映されるまで、数十秒〜数分かかることがあります。

---

以上で1曲分の「コーディング → ローカル確認 → 公開」が完了です。
2曲目以降も、同じ手順を繰り返すだけです。
